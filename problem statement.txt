Step 4: Verify Data Integrity Using Blockchain Hashing : 

Data Integrity means:
Checking whether the data has been changed or not during transmission.
Blockchain Hashing means:
Creating a unique digital fingerprint (hash) for the data and storing/verifying it through a blockchain mechanism.

Simple example : 

Suppose a sensor sends: Soil Moisture = 20%
A hash is generated: Hash = ABC123 Now the data travels through the network or server .

Case 1: Data Not Changed
Received Data = Soil Moisture = 20%
Generated Hash = ABC123
Stored Hash = ABC123

Both hashes match ✅

Result: Data is original and trustworthy.

Case 2: Data Changed by Attacker
Attacker modifies: Soil Moisture = 80%
Now the new hash becomes:
Generated Hash = XYZ789
Stored Hash = ABC123
Hashes do not match ❌

Result: Someone tampered with the data.


1. What is Blockchain?

A blockchain is a chain of blocks where each block stores data and is linked to the previous block using a hash.

Simple Structure
Block 1
Data
Hash
      ↓
Block 2
Data
Previous Hash
Hash
      ↓
Block 3
Data
Previous Hash
Hash

Because every block contains the previous block's hash, changing one block breaks the chain.

In Your Project

The blockchain may store:

Sensor Data Hash
Timestamp
Device ID

instead of storing all sensor data.

Purpose

To make it very difficult for attackers to modify records without being detected.

2. What is Hashing?

Hashing converts data into a fixed-length fingerprint called a hash.

Example:

Data:
Soil Moisture = 20%

Hash:
A1B2C3D4

Even a tiny change produces a completely different hash.

Soil Moisture = 21%

Hash becomes:

X9Y8Z7K6
Properties
Same input → Same hash
Different input → Different hash
One-way process
Cannot recover original data
3. Why is Hashing Used in Blockchain?

Suppose:

Block 1 Hash = ABC123

Block 2 stores:

Previous Hash = ABC123

If an attacker changes Block 1:

Block 1 Hash = XYZ789

Now Block 2 still contains:

Previous Hash = ABC123

Mismatch occurs.

The blockchain immediately detects tampering.

Simple Idea

Blockchain uses hashes to link blocks together and detect modifications.

4. What is Encryption?

Encryption converts readable data into unreadable data.

Example:

Original:
Soil Moisture = 20%

After encryption:

X7P9K2M8

Nobody can understand it.

5. What is Decryption?

Decryption converts encrypted data back to the original form.

Encrypted Data
      ↓
Decryption Key
      ↓
Original Data

Example:

X7P9K2M8
      ↓
Decrypt
      ↓
Soil Moisture = 20%
6. Hashing vs Encryption
Feature	Hashing	Encryption
Purpose	Verify integrity	Protect confidentiality
Reversible?	No	Yes
Output	Hash value	Cipher text
Original data recovered?	No	Yes
Uses key?	No	Yes
Used in Blockchain?	Yes	Usually No
7. Advantages of Hashing
Advantages

✅ Very fast

✅ Detects data modification

✅ Fixed-size output

✅ Good for integrity verification

✅ Core technology in blockchain

Disadvantages

❌ Cannot recover original data

❌ Does not hide data

❌ If data is sent without encryption, others can still read it

8. Advantages of Encryption
Advantages

✅ Keeps data secret

✅ Protects privacy

✅ Original data can be recovered

✅ Prevents unauthorized reading

Disadvantages

❌ Requires encryption/decryption keys

❌ More computational overhead

❌ If the key is stolen, security is compromised
